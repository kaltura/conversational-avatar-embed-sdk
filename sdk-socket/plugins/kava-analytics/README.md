# Kaltura Avatar Analytics (KAVA Plugin)

Optional analytics plugin for the [Kaltura Avatar SDK (Socket)](../../README.md). Reports Immersive Agent lifecycle events and standard KAVA events to Kaltura's analytics endpoint.

---

## Installation

### CDN (recommended)

```html
<!-- After the core SDK -->
<script src="https://cdn.jsdelivr.net/gh/kaltura/conversational-avatar-embed-sdk@latest/sdk-socket/plugins/kava-analytics/kaltura-avatar-analytics.js"></script>
```

### Local

```html
<script src="plugins/kava-analytics/kaltura-avatar-analytics.js"></script>
```

## Quick Start

```javascript
const sdk = new KalturaAvatarSDK({ clientId: '...', flowId: '...', container: '#avatar' });

const kava = new KalturaAvatarAnalytics(sdk, {
  ks: 'your-kaltura-session-token',   // Required — see KS_GUIDE.md
  partnerId: 12345                     // Required — your Kaltura partner ID
});

sdk.connect();
// Plugin auto-reports: callStarted → messageResponse (per speech) → callEnded
```

## What's Tracked Automatically

| SDK Event | KAVA Event | Code | Description |
|-----------|-----------|------|-------------|
| `ready` | callStarted | 80002 | Avatar session began |
| `avatar-speech` | messageResponse (experience=2) | 80001 | Avatar spoke |
| `user-speech` (final only) | messageResponse (experience=1) | 80001 | User spoke |
| `disconnected` / page unload | callEnded (with duration + error if any) | 80003 | Session ended |
| `reconnected` | callEnded + new callStarted | 80003 → 80002 | New thread, fresh state |
| `error` | *(captured, enriches callEnded)* | — | SDK error stored for callEnded |

## Configuration

```javascript
new KalturaAvatarAnalytics(sdk, {
  // ── Required ──
  ks: string,           // Kaltura Session token (app-provided, never generated by plugin)
  partnerId: number,    // Kaltura partner ID

  // ── Optional (defaults shown) ──
  agentId: '',          // Default: auto-read from sdk.getFlowId()
  genieId: '',          // Default: auto-read from sdk.getClientId()
  hostingApplication: 29,  // Numeric enum — see HostingApplication constants below
  hostingApplicationVer: '1.0.0',  // Default: plugin version
  clientTag: '',        // e.g., 'my-app:1.0.0' — identifies your app in reports
  serviceUrl: 'https://analytics.kaltura.com/api_v3/index.php',
  requestMethod: 'POST',
  userId: '',           // User ID for attribution (resolved to kuserId server-side)

  // ── Behavior ──
  autoStart: true,      // Fire callStarted on 'ready' event
  autoEnd: true,        // Fire callEnded on disconnect/unload/destroy
  autoMessages: true,   // Fire messageResponse on avatar-speech / user-speech
  beaconOnUnload: true, // Use sendBeacon for page-leave events (reliable on mobile)

  // ── Hooks ──
  tamperHandler: null,  // fn(params) => boolean — return false to suppress the event

  debug: false          // Log all events to console
});
```

## API Reference

### Manual Events

```javascript
kava.sendFeedback(messageId, 'like' | 'dislike');  // 80005 — messageFeedback
kava.pageView(pageName, pageType);                  // 10003 — pageLoad
kava.buttonClick(name, type, value);                // 10002 — buttonClicked
kava.customEvent(eventType, { ...extraFields });    // any event type
```

**Input validation:** All public methods silently no-op on invalid input (falsy required params, wrong types). The plugin never throws from public methods after construction.

### Context (affects all subsequent events)

```javascript
kava.setContextId('3:earnings');     // e.g., slideNum:category (max 50 chars)
kava.setContextType(1);              // 1=Entry, 2=Channel, 3=Global
kava.setMetadata('customVar1', 'value');
kava.setKS('new-ks-token');          // Update KS mid-session (takes effect on next event)
```

### Observability

```javascript
// Subscribe to plugin events
const unsubscribe = kava.on('sent', ({ eventType, params }) => { ... });
kava.on('error', ({ eventType, error }) => { ... });

// Unsubscribe
unsubscribe();                       // via returned function
kava.off('sent', handler);           // via explicit off()

// Stats snapshot
kava.getStats();
// → { eventsSent, errors, transportErrors, messageCount, callDuration,
//      lastError, threadId, sessionId }
```

### Lifecycle

```javascript
kava.destroy();  // Fires callEnded if active, unsubscribes all handlers, removes unload listener
```

Calling `destroy()` multiple times is safe (no-op after first call). After destroy, all public methods silently no-op.

## Error Handling & Resilience

The plugin is designed to never affect the hosting application or SDK:

- **All SDK event handlers** are wrapped in try/catch — a handler error is counted in stats but never propagates
- **Transport failures** (fetch/sendBeacon) are counted separately in `transportErrors` — no retry, no throw
- **`tamperHandler` exceptions** are caught and logged (the event is still sent)
- **SDK errors** are captured and enriched into the `callEnded` event (errorCode, errorMessage, errorPosition)
- **No blocking** — all fetch calls use `keepalive: true`; the plugin never blocks the main thread
- **No global state** — multiple plugin instances can coexist on different SDK instances without interference

## KS Responsibility

The app provides the KS — this plugin never generates, refreshes, or validates it. If the app refreshes the KS mid-session, call `kava.setKS(newKS)`.

See **[KS_GUIDE.md](./KS_GUIDE.md)** for:
- Which KS type to use (Widget Session recommended)
- How to generate it (client-side and server-side)

- Security considerations

## Transport

| Scenario | Method | Why |
|----------|--------|-----|
| Normal events | `fetch()` with `keepalive: true` (HTTP POST) | Ensures completion even during navigation |
| Page-leave events | `navigator.sendBeacon()` on `visibilitychange: hidden` | Reliable on mobile (fires before full unload) |

No retry logic, no batching — events fire immediately. Avatar conversations have low event frequency (~1-2 events/minute typical), so batching adds complexity without benefit.

## Verifying Events in Kaltura Reports

Events are processed by the analytics pipeline and appear in reports after aggregation (typically under 1 hour).

### Report Types

| Report ID | Name | Key Metrics |
|-----------|------|-------------|
| 80001 | Highlights | unique_threads, messageResponse, avg_messages, unique_known_users |
| 80002 | Messages Overtime | messageResponse count over time |
| 80003 | Message Feedback | messageFeedbackSent count |
| 80004 | Top Sources | messageResponse by source |
| 80005 | Avatar Sessions | callStarted, call duration, avg duration |
| 80006 | Experience Types | breakdown by Chat (user) vs Call (avatar) |

### Query Example

```bash
# Generate an admin KS (server-side only — never expose admin secret to clients)
KS=$(curl -s -X POST "https://www.kaltura.com/api_v3/service/session/action/start" \
  -d "secret=YOUR_ADMIN_SECRET" \
  -d "partnerId=YOUR_PARTNER_ID" \
  -d "type=2" \
  -d "userId=YOUR_USER_ID" \
  -d "format=1" | tr -d '"')

# Query Avatar Sessions report (last 7 days)
curl -s -X POST "https://www.kaltura.com/api_v3/service/report/action/getTotal" \
  -d "ks=${KS}" \
  -d "reportType=80005" \
  -d "reportInputFilter[fromDate]=$(date -v-7d +%s)" \
  -d "reportInputFilter[toDate]=$(date +%s)" \
  -d "reportInputFilter[objectType]=KalturaReportInputFilter" \
  -d "format=1"
```

### Expected Response

```json
{
  "header": "callStarted,avatar_call_duration,avatar_avg_call_duration",
  "data": "8,6.85,0.85625",
  "objectType": "KalturaReportTotal"
}
```

## Integration Example

```javascript
const sdk = new KalturaAvatarSDK({
  clientId: 'YOUR_CLIENT_ID',
  flowId: 'agent-24',
  container: '#avatar'
});

// Generate a Widget KS (see KS_GUIDE.md for production patterns)
const ks = await getWidgetKS(YOUR_PARTNER_ID);

const kava = new KalturaAvatarAnalytics(sdk, {
  ks: ks,
  partnerId: YOUR_PARTNER_ID,
  clientTag: 'my-app:2.0.0',
  hostingApplication: KalturaAvatarAnalytics.HostingApplication.AGENTIC_AVATARS_STUDIO
});

// Track user interactions
document.getElementById('like-btn').onclick = () => {
  kava.sendFeedback(currentMessageId, 'like');
};

// Track navigation
function goToSlide(num, title) {
  kava.setContextId(num + ':' + getCategory(num));
  kava.pageView('Slide ' + num + ' - ' + title, String(num));
}

// Monitor analytics health
kava.on('error', ({ eventType, error }) => {
  console.warn('Analytics failed for event', eventType, error.message);
});

sdk.connect();
```
